📝 The Complete MDTDF Prompt

ℹ️ About This Prompt

This is the complete, production-ready prompt for generating comprehensive multi-dimensional taxonomies. It has been refined through hundreds of iterations and produces consistent, high-quality taxonomic structures.

You are an expert taxonomist specializing in multi-dimensional classification systems. Your task is to create a comprehensive, orthogonal taxonomy for structured analysis.

DOMAIN: [INSERT YOUR DOMAIN HERE]

CONTEXT: [OPTIONAL: Add specific context about your research focus, data types, or analytical goals]

Create a Multi-Dimensional Taxonomy Development Framework (MDTDF) with the following specifications:

## STRUCTURAL REQUIREMENTS

Generate exactly 9 High-Level Taxonomic Principles (HLTPs) where:
- Each HLTP represents a completely orthogonal dimension of analysis
- Each HLTP must be independent (changing position on one dimension doesn't affect others)
- Each HLTP addresses a fundamentally different aspect of the domain

For each HLTP, create:
- Exactly 4 Second-Level Taxonomic Elements (L2)
- For each L2, exactly 3 Third-Level Taxonomic Elements (L3)
- For each L3, exactly 8 specific Taxa

This creates 9 × 4 × 3 × 8 = 864 unique classification points.

## ORTHOGONALITY REQUIREMENTS

Ensure dimensional independence by verifying:
1. Each HLTP can vary independently of all others
2. An item can be "high" on dimension A while being "low" on dimension B
3. No logical dependencies exist between dimensions
4. Each dimension adds unique classificatory power

## NAMING CONVENTIONS

HLTPs (Dimensions):
- Use clear, descriptive names (e.g., "Temporal", "Geographic", "Methodological")
- Avoid overlapping conceptual territories
- Maintain consistent abstraction level

L2 Elements (Major Categories):
- Name as subcategories that exhaustively partition the HLTP
- Use parallel grammatical structure within each HLTP
- Ensure MECE (Mutually Exclusive, Collectively Exhaustive)

L3 Elements (Subcategories):
- Provide finer granularity while maintaining clarity
- Keep names concise but specific
- Ensure logical hierarchy from L2

Taxa (Specific Elements):
- Use concrete, observable descriptors
- Include enough detail for unambiguous classification
- Maintain consistent specificity level across all taxa

## OUTPUT FORMAT

Provide the taxonomy in Tab-Separated Values (TSV) format with exactly 4 columns:

HLTP[tab]2nd Level TE[tab]3rd Level TE[tab]Taxon

Begin with the header row, then provide all 864 data rows.

## QUALITY CRITERIA

Your taxonomy must:
✓ Cover the entire conceptual space of the domain
✓ Avoid redundancy between dimensions
✓ Enable precise classification of any item in the domain
✓ Support quantitative analysis and visualization
✓ Maintain consistent granularity at each level
✓ Use terminology accessible to domain experts
✓ Allow for edge cases and emerging phenomena

## COMMON DIMENSIONS TO CONSIDER

Adapt these to your specific domain:
1. Temporal (time-based aspects)
2. Geographic/Spatial (location or space-based)
3. Thematic/Topical (subject matter)
4. Methodological (approaches, techniques)
5. Stakeholder/Actor (who is involved)
6. Functional/Purpose (why/what for)
7. Process/Lifecycle (stages, phases)
8. Impact/Outcome (effects, results)
9. Resource/Medium (tools, channels)

Generate the complete taxonomy now. Begin with the TSV header, followed by all 864 rows.

🏗️ Understanding the Structure

The Mathematical Foundation

Why 9 × 4 × 3 × 8?

This specific structure balances several competing needs:

  • 9 dimensions: Comprehensive coverage without cognitive overload
  • 4 L2 categories: Natural quartile divisions or 2×2 matrices
  • 3 L3 subcategories: Manageable refinement (low/medium/high, past/present/future, etc.)
  • 8 taxa: Sufficient specificity for detailed classification

Total: 864 unique classification points - enough for nuanced analysis while remaining manageable.

Hierarchical Levels Explained

Level Name Count Purpose Example
1 HLTP (Dimension) 9 Independent analytical axes Geographic
2 L2 (Major Category) 36 total (4 per HLTP) Primary divisions within dimension Regional, National, International, Global
3 L3 (Subcategory) 108 total (3 per L2) Refined classifications Sub-regional, Cross-border, Transnational
4 Taxa (Elements) 864 total (8 per L3) Specific, observable items Baltic cooperation, Nordic council, etc.

Orthogonality Principle

⚠️ Critical Requirement

Each dimension must be truly independent. Test this by asking: "Can an item be high on dimension A while being low on dimension B?" If the answer is no, the dimensions are not orthogonal.

Good Example:

  • Temporal dimension (when) ⊥ Geographic dimension (where)
  • A recent event (high temporal) can be local (low geographic)

Bad Example:

  • "Complexity" and "Sophistication" - these overlap conceptually
  • High complexity usually implies high sophistication

🔧 Processing Your Taxonomy

Python Processing Script

import pandas as pd
import numpy as np
import json
from collections import defaultdict

def load_taxonomy(file_path):
    """Load taxonomy from TSV file"""
    df = pd.read_csv(file_path, sep='\t')
    return df

def validate_structure(df):
    """Validate the taxonomy structure"""
    print("🔍 Validating Taxonomy Structure...")

    # Check column names
    expected_cols = ['HLTP', '2nd Level TE', '3rd Level TE', 'Taxon']
    if list(df.columns) != expected_cols:
        print(f"⚠️ Warning: Column names don't match expected: {expected_cols}")

    # Count unique values at each level
    n_hltps = df['HLTP'].nunique()
    n_l2s = df.groupby('HLTP')['2nd Level TE'].nunique().mean()
    n_l3s = df.groupby(['HLTP', '2nd Level TE'])['3rd Level TE'].nunique().mean()
    n_taxa = len(df)

    print(f"📊 Structure Summary:")
    print(f"  - HLTPs: {n_hltps} (expected: 9)")
    print(f"  - Avg L2s per HLTP: {n_l2s:.1f} (expected: 4)")
    print(f"  - Avg L3s per L2: {n_l3s:.1f} (expected: 3)")
    print(f"  - Total Taxa: {n_taxa} (expected: 864)")

    # Check for completeness
    if n_taxa == 864:
        print("✅ Taxonomy is complete!")
    else:
        print(f"⚠️ Taxonomy has {n_taxa} taxa, expected 864")

    return n_hltps, n_l2s, n_l3s, n_taxa

def check_orthogonality(df):
    """Check for potential overlaps between dimensions"""
    print("\n🔍 Checking Orthogonality...")

    hltps = df['HLTP'].unique()
    taxa_by_hltp = {}

    for hltp in hltps:
        taxa = set(df[df['HLTP'] == hltp]['Taxon'].str.lower())
        taxa_by_hltp[hltp] = taxa

    overlaps = []
    for i, hltp1 in enumerate(hltps):
        for hltp2 in hltps[i+1:]:
            common = taxa_by_hltp[hltp1] & taxa_by_hltp[hltp2]
            if common:
                overlaps.append((hltp1, hltp2, common))

    if overlaps:
        print("⚠️ Potential overlaps detected:")
        for hltp1, hltp2, common in overlaps[:3]:  # Show first 3
            print(f"  {hltp1} ↔ {hltp2}: {len(common)} common terms")
            print(f"    Examples: {list(common)[:3]}")
    else:
        print("✅ No obvious overlaps detected between dimensions")

def export_for_visualization(df, output_prefix='taxonomy'):
    """Export taxonomy in various formats for visualization"""
    print(f"\n📤 Exporting taxonomy...")

    # 1. Hierarchical JSON for D3.js sunburst
    hierarchy = {"name": "Taxonomy", "children": []}

    for hltp in df['HLTP'].unique():
        hltp_node = {"name": hltp, "children": []}
        hltp_df = df[df['HLTP'] == hltp]

        for l2 in hltp_df['2nd Level TE'].unique():
            l2_node = {"name": l2, "children": []}
            l2_df = hltp_df[hltp_df['2nd Level TE'] == l2]

            for l3 in l2_df['3rd Level TE'].unique():
                l3_node = {"name": l3, "children": []}
                l3_df = l2_df[l2_df['3rd Level TE'] == l3]

                for taxon in l3_df['Taxon'].unique():
                    l3_node["children"].append({"name": taxon, "value": 1})

                l2_node["children"].append(l3_node)
            hltp_node["children"].append(l2_node)
        hierarchy["children"].append(hltp_node)

    with open(f'{output_prefix}_hierarchy.json', 'w') as f:
        json.dump(hierarchy, f, indent=2)
    print(f"  ✓ Saved hierarchical JSON: {output_prefix}_hierarchy.json")

    # 2. Flat CSV for Excel/Sheets
    df.to_csv(f'{output_prefix}_flat.csv', index=False)
    print(f"  ✓ Saved flat CSV: {output_prefix}_flat.csv")

    # 3. Dimension summary
    summary = []
    for hltp in df['HLTP'].unique():
        hltp_df = df[df['HLTP'] == hltp]
        summary.append({
            'Dimension': hltp,
            'L2_Count': hltp_df['2nd Level TE'].nunique(),
            'L3_Count': hltp_df['3rd Level TE'].nunique(),
            'Taxa_Count': len(hltp_df),
            'Example_Taxa': ', '.join(hltp_df['Taxon'].head(3).tolist())
        })

    summary_df = pd.DataFrame(summary)
    summary_df.to_csv(f'{output_prefix}_summary.csv', index=False)
    print(f"  ✓ Saved dimension summary: {output_prefix}_summary.csv")

def generate_coding_template(df, output_file='coding_template.xlsx'):
    """Generate an Excel template for manual coding"""
    print(f"\n📝 Generating coding template...")

    # Create a matrix where rows are items to code and columns are taxa
    template_data = {'Item_ID': [], 'Item_Description': []}

    # Add all taxa as columns
    for _, row in df.iterrows():
        col_name = f"{row['HLTP']}_{row['2nd Level TE']}_{row['3rd Level TE']}_{row['Taxon']}"
        template_data[col_name] = []

    # Add some example rows
    for i in range(10):
        template_data['Item_ID'].append(f'ITEM_{i+1:03d}')
        template_data['Item_Description'].append(f'[Description of item {i+1}]')
        for col in template_data.keys():
            if col not in ['Item_ID', 'Item_Description']:
                template_data[col].append(0)  # 0 = not applicable, 1 = applicable

    template_df = pd.DataFrame(template_data)
    template_df.to_excel(output_file, index=False)
    print(f"  ✓ Saved coding template: {output_file}")

# Main execution
if __name__ == "__main__":
    # Load your taxonomy
    df = load_taxonomy('taxonomy.tsv')

    # Validate structure
    validate_structure(df)

    # Check orthogonality
    check_orthogonality(df)

    # Export for various uses
    export_for_visualization(df)

    # Generate coding template
    generate_coding_template(df)

    print("\n🎉 Taxonomy processing complete!")

validate_structure <- function(df) {
  cat("🔍 Validating Taxonomy Structure...\n")

  n_hltps <- n_distinct(df$HLTP)

  n_taxa <- nrow(df)

  cat(sprintf("  - HLTPs: %d (expected: 9)\n", n_hltps))
  cat(sprintf("  - Avg L2s per HLTP: %.1f (expected: 4)\n", n_l2s))
  cat(sprintf("  - Total Taxa: %d (expected: 864)\n", n_taxa))

  if (n_taxa == 864) {
    cat("✅ Taxonomy is complete!\n")
  } else {
    cat(sprintf("⚠️ Taxonomy has %d taxa, expected 864\n", n_taxa))
  }
}

# Create visualization data
create_viz_data <- function(df) {
  # Sunburst data
  sunburst_data <- df %>%
    group_by(HLTP, `2nd Level TE`, `3rd Level TE`) %>%
    summarise(count = n(), .groups = 'drop')

  # Dimension summary
  dimension_summary <- df %>%
    group_by(HLTP) %>%
    summarise(
      L2_Count = n_distinct(`2nd Level TE`),
      L3_Count = n_distinct(`3rd Level TE`),
      Taxa_Count = n(),
      .groups = 'drop'
    )

  return(list(sunburst = sunburst_data, summary = dimension_summary))
}

Common Issues & Solutions

⚠️ Troubleshooting

Issue: Dimensions seem to overlap

  • Solution: Reframe dimensions to focus on truly different aspects
  • Example: Instead of "Complexity" and "Sophistication", use "Technical Complexity" and "Resource Requirements"

Issue: Can't find 8 taxa for some L3s

  • Solution: Either broaden the L3 category or add emerging/potential items
  • Consider: Historical, current, and future possibilities

Issue: Some dimensions don't apply to all items

  • Solution: This is OK! Use "Not Applicable" as a valid classification
  • The dimension should still be relevant to the domain overall
